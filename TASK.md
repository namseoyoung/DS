# 개발 순서

이 프로젝트는 한 번에 모든 기능을 구현하지 않고 단계별로 구현한다.

## Phase 1. 프로젝트 초기 세팅

상태: 완료

- React + TypeScript + Tailwind CSS 세팅
- Express 서버 세팅
- Supabase 연결 준비
- 환경변수 파일 구성
- `/participant`, `/admin`, `/display` 라우팅 생성
- README 작성

## Phase 2. DB 스키마 구현

상태: 완료

- `users`
- `companies`
- `investments`
- `transactions`
- `game_status`
- `company_value_history`
- `news`
- `announcements`
- `connection_status`

완료 기준:

- 초기 테스트 데이터 생성
- 5개 기업 데이터 생성
- 30~40명 테스트 계정 생성 가능

## Phase 3. 로그인 기능

상태: 완료

- 로그인 페이지
- 아이디/비밀번호 검증
- 참가자 로그인 시 `/participant` 사용
- 관리자 로그인 시 `/admin` 사용
- 로그인 사용자 정보 유지

## Phase 4. 참가자 페이지

상태: 완료

- 본인 자산 요약 카드
- 보유 현금, 평가액, 총자산, 수익률 표시
- 현재 연차/상태 안내
- 본인 보유 투자 목록
- 기업 카드 목록
- 투자 모달
- 공지 토스트
- 그래프 하단 기업별 현재 가치/변동률 표시

## Phase 5. 투자 기능

상태: 완료

- 투자 API 구현
- cash 감소
- investment 증가
- transactions 로그 저장
- 참가자/관리자/전광판 자동 갱신

## Phase 6. 관리자 운영 대시보드

상태: 완료

- 현재 연차, 상태, 남은 시간, 접속 인원 표시
- 연봉 지급, 투자 시작/마감, 정산, 다음 연차, 일시정지/재개 버튼
- 투자 마감 시 참가자 투자 버튼 비활성화

## Phase 7. 회원/기업 관리

상태: MVP 완료

- 회원 목록 테이블
- 회원별 자산 확인
- 회원 닉네임, 실명, 직급, 소속 회사, 현금 수정
- 기업명, 현재 가치, 초기 자본 수정

다음 개선:

- prompt 기반 수정 UI를 전용 모달/인라인 폼으로 교체

## Phase 8. 연봉 지급

상태: 완료

- 직급별 연봉 로직 구현
- 기본 4개 기업 연봉표 적용
- 예일회사 연봉표 적용
- 연봉 지급 로그 저장

## Phase 9. 1~3년차 정산

상태: 완료

- 기업별 변동률 입력 UI
- 정산 확정 버튼
- 평가금액/총자산/랭킹 서버 계산

## Phase 10. 뉴스/공지 시스템

상태: MVP 완료

- 뉴스 등록
- 공지 등록
- 참가자 화면 표시
- 관리자 빠른 공지 버튼
- 전광판 최신 뉴스/공지 표시
- 뉴스/공지 저장

다음 개선:

- 참가자 화면 토스트/모달 알림 효과 추가

## Phase 11. 4년차 실시간 그래프

상태: 완료

- `company_value_history` 사용
- Recharts 선 그래프 구현
- 기업별 가치 히스토리 저장
- 투자금 기준 기업 가치 변동 로직 구현
- 관리자 수동 갱신 버튼 구현
- `REALTIME_ROUND` 상태에서 서버가 10초마다 자동 가치 갱신

## Phase 12. 자동 카운트다운

상태: MVP 완료

- 관리자 투자 시간 설정
- 서버 카운트다운
- 1분 전 자동 공지
- 30초 전 자동 공지
- 마지막 10초 전광판 카운트다운
- 0초 시 투자 자동 마감
- 자동 정산 실행
- 일시정지 시 남은 시간 보존, 재개 시 타이머 복원

다음 개선:

- 자동 정산 시 4년차 최종 순위별 변동 규칙 적용

## Phase 13. 전광판 모드

상태: 완료

- 현재 연차, 남은 시간, 기업 TOP5, 개인 TOP5
- 최신 뉴스, 최근 투자 로그
- 마지막 10초 카운트다운
- 종료 후 엔딩 화면

## Phase 14. 최종 엔딩

상태: 완료

- 기업 최종 순위 계산
- 개인 최종 순위 계산
- 최종 우승 기업 화면
- 개인 자산 1등 화면

## Phase 15. 배포 준비

상태: 준비 문서 완료, 실제 배포 미진행

- Vercel 배포 설정
- Render/Railway 백엔드 배포 설정
- Supabase 환경변수 설정
- CORS 설정

남은 작업:

- 실제 모바일 접속 테스트
- 30~40명 동시 접속 리허설
- localhost가 아닌 배포 URL 검증
